# Contributing

## Boundary Rules

Before adding a document, registry entry, or governance note, answer:

1. Is this an organizational map or is it detailed canonical implementation content?
2. Which repo already owns the source of truth?
3. Should this repo summarize and link, or should the change happen in the owning repo instead?

### Put it in `OpenSIN-overview` if:
- it clarifies repo roles
- it defines ownership boundaries
- it indexes teams, agents, MCPs, websites, or templates
- it helps people find the correct canonical repo

### Do NOT put it in `OpenSIN-overview` if:
- it is the detailed docs canon
- it is runtime implementation truth
- it is product or control-plane implementation truth
- it is canonical OpenCode config content
